# Static Segment

```sh npm2yarn
npx vovk new segment openapi --static # will create a new static segment with name "openapi" at src/app/api/openapi/[[...vovk]]/route.ts
```

Next.js can render API endpoints at build time using [generateStaticParams](https://nextjs.org/docs/app/api-reference/functions/generate-static-params). Vovk.ts provides the `generateStaticAPI` function to utilize this feature and generate static API endpoints for the lowest possible latency. You can use it to serve OpenAPI data, historical data (you can configure a CI/CD job to run periodically to update it), or any other static data that doesn't change often. It can also be used in [Static Export mode](https://nextjs.org/docs/pages/building-your-application/deploying/static-exports) with the `output: 'export'` Next.js config setting:

```ts filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  output: 'export',
};

module.exports = nextConfig;
```

The only thing you need to do is implement `generateStaticParams` that returns `generateStaticAPI` with the controller list.

```ts filename="src/app/api/[[...vovk]]/route.ts"
// ...
export type Controllers = typeof controllers;

export function generateStaticParams() {
  return generateStaticAPI(controllers);
}

export const { GET } = initSegment({ controllers });
```

To make it work on static website hosting like GitHub Pages, you may need to define the `.json` extension in your endpoint definition to ensure proper HTTP headers are returned.

```ts
import { get, prefix } from 'vovk';

@prefix('hello')
export default class HelloController {
  @get('greeting.json')
  static async getHello() {
    return { greeting: 'Hello world!' };
  }
}
```

As a result, you'll get an endpoint that looks like this: [https://vovk.dev/api/hello/greeting.json](https://vovk.dev/api/hello/greeting.json) (hosted on GitHub Pages).

If you use a custom slug (e.g., `/src/app/api/[[...custom]]/route.ts`) instead of `vovk`, the default value, you can provide it as a second argument.

```ts
export function generateStaticParams() {
  return generateStaticAPI(controllers, 'custom');
}
```

## Static Endpoint Parameters

The `@get` decorator accepts a set of options, one of which is `staticParams`, which allows you to define static parameters for the endpoint to simulate conditional routing. The example below demonstrates a static API represented by a single handler that renders 6 sets of static parameters: section (`a | b`) and page (`1 | 2 | 3`).

```ts
import { z } from 'zod/v4';
import { prefix, get, operation } from 'vovk';
import { withZod } from 'vovk-zod';

@prefix('static-params')
export default class StaticParamsController {
  @operation({
    summary: 'Static Params',
    description: 'Get the static params: section and page',
  })
  @get('{section}/page{page}.json', {
    staticParams: [
      { section: 'a', page: '1' },
      { section: 'a', page: '2' },
      { section: 'a', page: '3' },
      { section: 'b', page: '1' },
      { section: 'b', page: '2' },
      { section: 'b', page: '3' },
    ],
  })
  static getStaticParams = withZod({
    params: z.object({
      section: z.enum(['a', 'b']),
      page: z.enum(['1', '2', '3']),
    }),
    handle: async (_req, { section, page }) => {
      return { section, page };
    },
  });
}
```

It will build 6 JSON files:

- [/static-params/a/page1.json](https://vovk-examples.vercel.app/api/static/static-params/a/page1.json)
- [/static-params/a/page2.json](https://vovk-examples.vercel.app/api/static/static-params/a/page2.json)
- [/static-params/a/page3.json](https://vovk-examples.vercel.app/api/static/static-params/a/page3.json)
- [/static-params/b/page1.json](https://vovk-examples.vercel.app/api/static/static-params/b/page1.json)
- [/static-params/b/page2.json](https://vovk-examples.vercel.app/api/static/static-params/b/page2.json)
- [/static-params/b/page3.json](https://vovk-examples.vercel.app/api/static/static-params/b/page3.json)

See the live example [here](https://vovk-examples.vercel.app/static-params).

## RPC Client

It doesn't matter whether you use a static API or a dynamic one; you can use the same RPC client to call it. This includes client-side validation, type inference, etc.

```ts
const resp = await StaticParamsRPC.getStaticParams({
  params: {
    section: 'a',
    page: '1',
  },
});

console.log(resp); // { section: 'a', page: '1' }
```

## Troubleshooting

Sometimes when `output: 'export'` is set in `next.config.js`, you may see a schema parsing error when the `dev` command is run:

```
 тип SyntaxError: Unexpected non-whitespace character after JSON at position 396 (line 1 column 397)
    at JSON.parse (<anonymous>) {
  page: '/api/_schema_'
}
```

To avoid this error when running `npm run dev`, close the browser tabs where the app is opened until the initial request to `/_schema_` is finished. After that, you can open the app in the browser again, and it should work fine. If you still see the error, feel free to open an issue at the [Vovk.ts GitHub repository](https://github.com/finom/vovk) with the details.
