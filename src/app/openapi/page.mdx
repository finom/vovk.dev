# OpenAPI

Vovk can generate OpenAPI 3.1 schema from your controllers and export it as a JSON file with **vovk-openapi** package that exports `openapi` decorator.

<Tabs items={['npm', 'pnpm', 'yarn']}>
  <Tabs.Tab>
``` 
npm i vovk-openapi 
```
</Tabs.Tab>
  <Tabs.Tab>
``` 
pnpm i vovk-openapi 
```
</Tabs.Tab>
  <Tabs.Tab>
```
yarn add vovk-openapi
```
</Tabs.Tab>
</Tabs>

```ts filename="src/modules/user/UserController.ts"
import { get, post, prefix } from 'vovk';
import { openapi } from 'vovk-openapi';

@prefix('users')
export default class UserController {
  @openapi({
    summary: 'Get user',
    description: 'Get user by id',
  })
  @get()
  static getUser() {
    // ...
  }
}
```

When handler is using any of the [validation libraries](/validation), the schema for body, query, params and output is generated automatically. You can also use `@openapi` decorator to provide custom schema for the request and response.
