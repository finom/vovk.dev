5:Tec5,# Packages and Repos

## Packages of the [Main Monorepo](https://github.com/finom/vovk):

### [vovk](https://www.npmjs.com/package/vovk)

The core library providing decorators, utilities, types, and other features used by both server-side and client-side code. It depends only on a few type-focused libraries and is reported as 100% self-composed on [Bundlephobia](https://bundlephobia.com/package/vovk).

```sh npm2yarn
npm install vovk@draft
```

### [vovk-cli](https://www.npmjs.com/package/vovk-cli)

The [CLI](/cli) package. Install globally or as a dev dependency. Provides the `vovk` binary.

```sh npm2yarn
npm install vovk-cli@draft --save-dev
```

```sh npm2yarn
npx vovk <command>
```

### [vovk-client](https://www.npmjs.com/package/vovk-client)

The client package that re-exports the [composed](/composed) [TypeScript](/typescript) client, typically located in `node_modules/.vovk-client`. Import it from client-side code (Node.js or browser) to access RPC methods and schemas.

```sh npm2yarn
npm install vovk-client@draft
```

### [vovk-init](https://www.npmjs.com/package/vovk-init)

A shortcut for the `vovk-cli init` command that initializes a Vovk.ts project with a [config](/config).

```sh npm2yarn
npx vovk-init
```

### [vovk-zod](https://www.npmjs.com/package/vovk-zod)

The [Zod](/validation/zod) integration. Exposes `withZod` for validating HTTP requests on the server with Zod 4, and provides a Zod 3-compatible `withZod` available via `vovk-zod/v3`.

```sh npm2yarn
npm install vovk-zod@draft
```

### [vovk-dto](https://www.npmjs.com/package/vovk-dto)

The [DTO](/validation/dto) integration. Provides `withDto` for validating HTTP requests on the server with [class-validator](https://www.npmjs.com/package/class-validator), and `validateOnClient` (via `vovk-dto/validateOnClient`) to validate data on the client with class-validator instead of [Ajv](/validation/client#ajv).

```sh npm2yarn
npm install vovk-dto@draft
```

### [vovk-yup](https://www.npmjs.com/package/vovk-yup)

The [Yup](/validation/yup) integration. Provides `withYup` for validating HTTP requests on the server.

```sh npm2yarn
npm install vovk-yup@draft
```

### [vovk-ajv](https://www.npmjs.com/package/vovk-ajv)

Exposes `validateOnClient` to validate input data on the client using the emitted JSON Schema.

```sh npm2yarn
npm install vovk-ajv@draft
```

### [vovk-python](https://www.npmjs.com/package/vovk-python)

Provides Python client templates and utilities to generate the [Python client library](/python).

```sh npm2yarn
npm install vovk-python@draft --save-dev
```

### [vovk-rust](https://www.npmjs.com/package/vovk-rust)

Provides Rust client templates and utilities to generate the [Rust client library](/rust).

```sh npm2yarn
npm install vovk-rust@draft --save-dev
```

## Other Repos and Packages

### [vovk-hello-world](https://github.com/finom/vovk-hello-world)

A minimal example of using Vovk.ts. See the ["Hello World"](/hello-world) page for details.

### [vovk-multitenant-example](https://github.com/finom/vovk-multitenant-example)

Demonstrates [multitenancy](/multitenant) with Vovk.ts.

### [vovk-examples](https://examples.vovk.dev/)

A collection of examples and proofs of concept.

### [dto-mapped-types](https://github.com/finom/dto-mapped-types)

A fork of [@nestjs/mapped-types](https://www.npmjs.com/package/@nestjs/mapped-types) without the [@nestjs/common](https://www.npmjs.com/package/@nestjs/common) dependency. Works in the Edge Runtime, imports `reflect-metadata/lite`, and includes fixes for Edge-specific issues.

### [vovk-perf-test](https://github.com/finom/vovk-perf-test)

[Overhead performance](/performance) tests for Vovk.ts.
