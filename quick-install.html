Quick Start

Quick Start

Create a Next.js app (App Router + TypeScript)

npx create-next-app@latest my-app --ts --app --src-dir
pnpm dlx create-next-app@latest my-app --ts --app --src-dir
yarn dlx create-next-app@latest my-app --ts --app --src-dir
bun x create-next-app@latest my-app --ts --app --src-dir
cd my-app

Initialize Vovk.ts

You will be prompted to answer a few questions. The CLI will install the selected dependencies, update npm scripts, enable experimentalDecorators in tsconfig.json, and create a config file.

npx vovk-cli@latest init
pnpm dlx vovk-cli@latest init
yarn dlx vovk-cli@latest init
bun x vovk-cli@latest init

  • vovk init

Generate a controller and a service

This command scaffolds UserController.ts and UserService.ts from built-in (customizable) templates under ./src/modules/user/ and updates the segmentâ€™s route.ts:

npx vovk new controller service user
pnpm dlx vovk new controller service user
yarn dlx vovk new controller service user
bun x vovk new controller service user
src/modules/user/UserController.ts
import { procedure, prefix, get, put, post, del, operation } from 'vovk';
import { z } from 'zod';
import UserService from './UserService';
 
@prefix('users')
export default class UserController {
  // ...
  @operation({
    summary: 'Get single user',
  })
  @get('{id}')
  static getSingleUser = procedure({
    params: z.object({
      id: z.string(),
    }),
    handle(_req, { id }) {
      return UserService.getSingleUser(id);
    },
  });
  // ...
}

Start the server and inspect an endpoint

The dev script runs the Next.js dev server and the Vovk.ts watcher in parallel. While running, Vovk.ts generates schema files in .vovk-schema/ (commit these) and the client library in node_modules/.vovk-client.

npm run dev
pnpm run dev
yarn dev
bun run dev

Create a React component to display data

Import the generated client library and call the RPC method:

src/app/page.tsx
'use client';
import { useEffect, useState } from 'react';
import type { VovkReturnType } from 'vovk';
import { UserRPC } from 'vovk-client';
 
export default function Home() {
  const [resp, setResp] = useState<VovkReturnType<typeof UserRPC.getSingleUser> | null>(null);
 
  useEffect(() => {
    void UserRPC.getSingleUser({ params: { id: '123' } }).then(setResp);
  }, []);
  return <pre>Response: {JSON.stringify(resp, null, 2)}</pre>;
}

You can alternatively use React QueryÂ  for request state management:

import { useQuery } from '@tanstack/react-query';
import { UserRPC } from 'vovk-client';
// ...
const { data, error, isLoading } = useQuery({
  queryKey: UserRPC.getSingleUser.queryKey(),
  queryFn: () => UserRPC.getSingleUser({ params: { id: '123' } }),
});

Deploy

The vovk init command adds a prebuild script so vovk generate runs automatically before next build. If you adjust scripts manually, run:

npx vovk generate
pnpm dlx vovk generate
yarn dlx vovk generate
bun x vovk generate
